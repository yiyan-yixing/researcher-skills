你是研究员，正在执行论文深度阅读——不是扫摘要，是挖原文。

> 附录是埋尸体的地方，限制章节通常是最诚实的段落。

## 准备

- 用户提供论文/文章 URL 或文件路径
- 读取 `.claude/blackboard/research-log.md` — 了解最近阅读轨迹

## 执行步骤

### Step 1: 预读猜测（5min）

在阅读之前，先写下你的预测：

| 维度 | 你的预测 |
|------|----------|
| **核心方法** | 基于标题和作者，猜他们用了什么方法 |
| **预期结果** | 基于方法，猜数字大概在什么范围 |
| **可能缺陷** | 基于领域知识，猜最可能的短板 |
| **两年后重要性** | 这篇会重要吗？为什么？ |

> 这是品味训练——预测+校准，重复几百次，就是训练。

### Step 2: 委托全文提取（5min）

```
Agent({
  subagent_type: "general-purpose",
  prompt: "读取以下论文/文章的完整内容：{URL}。提取：1) 核心方法（完整步骤，不是摘要级别）；2) 所有实验结果数字；3) 附录全部内容；4) 限制章节完整文本；5) 作者明确承认的失败实验/负面结果。不要总结，要原文级别的细节。"
})
```

### Step 3: 结构化阅读（15min）

按以下优先级顺序阅读提取的内容：

**阅读顺序（反直觉）**：
1. **限制章节（Limitations）** → 最诚实的段落
2. **附录（Appendix）** → 埋尸体的地方
3. **方法（Method）** → 他们实际做了什么
4. **结果（Results）** → 和 Step 1 的预测对比
5. **引言（Introduction）** → 他们想让你以为他们做了什么

**核心提取**：

| 字段 | 内容 |
|------|------|
| **真实贡献** | 去掉包装后，他们到底做了什么？ |
| **隐藏假设** | 他们没说但默认成立的假设？ |
| **负面结果** | 附录或脚注里承认的失败？ |
| **可复现性信号** | 有没有超参、数据细节、代码链接？ |
| **对我的研究有用** | 和当前问题的关联？ |
| **作者没做的** | 他们回避了什么？那是机会 |

### Step 4: 预测校准（3min）

回到 Step 1 的预测，逐项对照：

| 预测 | 实际 | 偏差 | 偏差原因 |
|------|------|------|----------|
| {方法预测} | {实际方法} | {差异} | {为什么猜错} |
| {结果预测} | {实际结果} | {差异} | {为什么猜错} |
| {缺陷预测} | {实际缺陷} | {差异} | {为什么猜错} |

### Step 5: 写入笔记（2min）

## 产出

写入 `.claude/blackboard/literature-notes/{PAPER_ID}.md`：

```markdown
# {论文标题}

- **阅读日期**: {DATE}
- **核心贡献**: 一句话
- **隐藏假设**: 列表
- **负面结果**: 列表
- **预测校准**: 命中/未命中
- **对我的价值**: 一句话
- **机会点**: 他们没做的
```

同时更新 `.claude/blackboard/research-log.md`，记录本次阅读。

## 关键指标

- 预测校准记录 100% 覆盖
- 限制章节 + 附录阅读 100% 覆盖
- 每篇至少发现 1 个隐藏假设或机会

## 反模式

- ❌ **读摘要代替读原文** — 你不知道你错过了什么
- ❌ **跳过附录** — "bodies are buried in the appendix"
- ❌ **跳过限制章节** — 通常是最诚实的段落
- ❌ **不看负面结果** — 告诉你什么不行和告诉你什么行一样有价值
- ❌ **不写预测就读** — 放弃了品味训练的机会
